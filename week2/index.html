Chapter 2 - JavaScript: Novice to Ninja

  • Using comments in your code is very important and good use of comments is a mark of a professional programmer verse a novice.
  • // and /* */ are used for comments in JavaScript.
  • Alway use semicolons when needed even if it doesn't cause error.
  • The const keyword is used when the variable will not change and the let keyword is used when the variable will change in the future.
  • Variables with a global scope should be used little if not at all to keep the code clean and to prevent bugs.
  • JavaScript variable names should follow the camelCase style.
  • Using wrappers, JavaScript provide function for primatives values even though they are not objects.
  • Template Literals use the backtick instead of quotes to define strings.
  • 0x represents hexadecimals and 0o represents octal literals and 0b represents binary numbers and e1 is exponental notation.
  • The range of numbers JavaScript can handle is -1.7976931348623157e+308 to 1.7976931348623157e+308 and the smallest number JavaScript can handle is 5e-324.
  • NaN === NaN returns false and so use .isNaN() function instead.

Chapter 3 - JavaScript: Novice to Ninja

  • Destructuring an array is the concept of taking values out of an array and presenting them as individual values.
  • The pop, push, shift, and unshift methods are useful in add and removing values to and from the array.
  • Use the join function for converting the array to a string.
  • The reverse and sort methods are useful in ordering an array. They also change the array permantly.
  • Sets are useful if you have a collection of data and you don't want to check to see if there is any duplicates.
  • Sets do not use an index.
  • A set can be converted into an array using the spread operator.
  • A memory leak occurs when a program retains references to values that can no longer be accessed in its memory. This can slow down the program and even break the system if it turns out really bad.
  • The weak set can be used to prevent memory leaks.
  • Maps are like other objects but are used primarly for storage and retrival and they are very effencent at that.
  • Maps can be converted into an array using the spread operator.
  • A weak map can be used to prevent memory leaks.
  • JavaScript has if else statements that are similar to the way they are used in C++.
  • JavaScript has loops that are similar to the way they are used in C++.
  • Sets and Maps have methods that let you loop through them.

Chapter 4 - JavaScript: Novice to Ninja

  • Function can be decleared using function declearation, function expression, or with the function constructor.
  • We should not repeat code in our programs but use functions to store blocks of code and call the function as much as we need to.
  • All functions return a value and if they don't return anything, then they actually return undefined.
  • Hoisting is moving all the variables from a function to the scope above it.
  • A callback is when a function is passed in as a parameter to another function.
  • Arrow functions can be used to declear a function.
